1. Overview
The application is an AI‑algorithm generator and Agentic dynamic crypto trading system consisting of:
- A frontend GUI (React/Next.js or similar)
- A Node.js backend running the agent runtime and MCP tools
- A set of specialized agents utilizing a workflow schema responsible for trading logic, risk enforcement, market data, and user interaction
- A local infrastructure layer binding all components together on localhost and free‑tier operation
The system supports both paper trading and real trading, with strict safety boundaries enforced by the backend.

6. Specialized Agents & Their Duties
Your architecture benefits from role‑specific agents, each with a narrow, well‑defined responsibility. This keeps the system safe, debuggable, and modular.
6.1 Manager Agent (User-Facing Agent)
Primary role: 
- Handles all user interaction
- High-level decision-making.
Duties:
- Interprets user instructions from the chat window
- Manages strategy parameters
- Requests market data from the Market Agent
- Requests trade execution from the Execution Agent
- Ensures risk rules are respected (via Risk Agent)
- Provides explanations, summaries, and reasoning to the user
Never directly places trades.

6.3 Market Agent (Data Specialist)
Primary role:
Provides real-time and historical market data.
Duties:
- Fetches live price data
- Retrieves order book snapshots
- Provides technical indicators
- Supplies historical candles for strategy logic
- Alerts Manager Agent of volatility or anomalies
Tool calls:
get_market_data, get_historical_data, get_orderbook

6.3 Risk Agent (Safety Gatekeeper)
Primary role:
Enforces all risk management rules before any trade is executed.
Duties:
- Validates trade size
- Checks leverage limits
- Ensures max daily loss not exceeded
- Confirms stop-loss/take-profit rules
- Blocks trades that violate constraints
- Logs all risk decisions
Tool calls:
validate_trade, check_risk_limits, calculate_position_size

6.4 Execution Agent (Trade Executor)
Primary role:
Executes trades only after Risk Agent approval.
Duties:
- Places market/limit orders
- Cancels orders
- Manages open positions
- Confirms fills and returns results
- Never acts without explicit instruction from Manager Agent
Tool calls:
place_trade, cancel_order, get_open_positions

6.5 Strategy Agent
If algorithmic trading is enabled:
Duties:
- Runs strategy logic
- Generates trade signals
- Sends signals to Manager Agent
- Never bypasses Risk Agent or Execution Agent
If disabled, it remains idle.

6.6 How Calls Are Made (Workflow Examples)User>Manager Agent>Risk Agent>Execution Agent
7 User-Initiated Trade
- User: “Buy 0.1 BTC now.”
- Manager Agent interprets intent.
- Manager Agent requests price from Market Agent.
- Manager Agent sends proposed trade to Risk Agent.
- Risk Agent approves or rejects.
- If approved, Manager Agent instructs Execution Agent.
- Execution Agent places the trade via backend MCP tool.
- Backend calls exchange API.
- Result returned to GUI.

8. Strategy-Initiated Trade (Algorithmic Mode)
- Strategy Agent detects a signal.
- Sends signal to Manager Agent.
- Manager Agent validates context (mode, user settings).
- Manager Agent sends trade to Risk Agent.
- Risk Agent approves or rejects.
- Execution Agent handles the trade.
- GUI receives update.

8.3 Market Data Request
- User: “What’s the current price of ETH?”
- Manager Agent → Market Agent → Backend → Exchange API
- Response returned to GUI.

The backend sends:

Communication channels:
- REST for initial data
- WebSockets for live updates
- Event streams for agent messages
